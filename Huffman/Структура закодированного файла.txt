Описание структуры закодированного файла

Файл, созданный в результате кодирования, состоит из трех основных частей: словаря Хаффмана, длины закодированных данных и закодированных данных.

1. Словарь Хаффмана
Словарь используется для декодирования данных. Он записывается следующим образом:
+ Количество символов в словаре (4 байта):
	Тип: int
	[Указывает, сколько различных символов содержится в словаре.]
+ Для каждого символа:
	Символ (2 байта):
		Тип: char
		[Сам символ.]
	Код символа (переменной длины):
		Тип: UTF-строка
		[Закодированный бинарный код для символа.]

2. Длина закодированных данных
Указывает точное количество битов в закодированных данных:
+ Длина закодированных данных (4 байта):
	Тип: int
	[Указывает количество битов, используемых для хранения закодированных данных. Это нужно, чтобы при чтении закодированных данных игнорировать лишние нули в последнем байте.]

3. Закодированные данные
Закодированные данные записываются в бинарном формате, побитово. Каждые 8 битов (1 байт) объединяются, и оставшиеся биты дополняются нулями в конце файла.
